  • Participated in knowledge transfer (KT) sessions on API and performance testing.
Challenges & Solutions as QA Engineer:

  • Unclear Requirements: Ask questions early, participate in requirement reviews, and use user stories for clarity.
  • Tight Deadlines: Prioritize high-risk test cases, automate repetitive tasks, and communicate risks to the team.
  • Frequent Requirement Changes: Adapt test cases quickly and work closely with developers.
  • Limited Test Data: Create realistic test data sets or request production-like data from Dev/DB teams.
  • Environment Issues: Coordinate with DevOps early, report issues immediately, and have backup environments if possible.
  • High Number of Bugs Late in Cycle: Perform early testing (shift-left), do risk-based testing, and focus on key functionalities.
  • Poor Communication Between Teams: Join daily stand-ups, use JIRA/Slack effectively, and document all findings clearly.
  • Lack of Domain Knowledge: Study the product, read documentation, ask domain experts, and take notes during demos.
  • Limited Time for Regression Testing: Automate regression cases, run smoke tests first, and continuously update regression suite.
